# MatReflect_NN 前端高密度与效率重构方案

## 1. 核心目标：从“消费级网页”到“专业级软件”

当前的 UI 虽然在视觉上具备现代感，但存在信息密度过低、空间浪费严重的问题。作为一个涉及高频参数调整、文件挑选、批量渲染和训练对比的专业工作台，界面需要像 VSCode、Blender 或 Houdini 那样，**把每一寸像素都留给数据和操作**。

本次重构旨在：
1. **提升同屏信息量 30% - 50%**，减少用户的无效滚动和鼠标移动距离。
2. **释放核心画廊区域**，让对比和检视占据绝对主导地位。
3. **消除视觉噪音**，弱化容器边框，强化数据本体。

---

## 2. 全局基础样式改造 (Global Typography & Spacing)

### 2.1 字体与行高压缩 (Typeset Compression)
- **正文字号**：从默认的网页字号（约 16px）缩小为专业软件标准的 **13px - 14px**。
- **行高 (Line-height)**：从 `1.55` 降至 **`1.3 - 1.4`**。
- **标题降级**：大幅缩减所有 `h2`, `h3` 的字号，取消所有不必要的巨大“英雄区 (Hero)”标题。标题仅用粗体和微调字号区分层级即可。
- **次要信息弱化**：文件大小、路径、时间戳等次要信息的字号降至 `11px - 12px`，颜色调淡。

### 2.2 空间与留白剥离 (Whitespace Reduction)
- **全局 Padding**：页面边缘留白从 `40px/20px` 统一降至 **`12px`**。
- **卡片内边距**：所有的 `.render-section`, `.models-section` 的 padding 从 `16px` 砍半至 **`8px`**。
- **元素间隙 (Gap)**：按钮、表单和列表项的 gap 从 `12px-18px` 降至 **`4px-8px`**。
- **圆角锐化 (Border-radius)**：将厚重的 `8px` 或 `999px`（药丸形）统一削减为更加锐利紧凑的 **`4px`**（或直接直角化），去除“玩具感”。

---

## 3. 表单与控件高密度重组 (Form Compactness)

### 3.1 紧凑型表单布局
- 取消 `Label`（标题）与 `Input`（输入框）的上下堆叠（上下结构最占垂直空间）。
- 改为**同行左右结构（Inline form）**：`[ Label: Input ]`，或将多个短输入框放在同一行。
- 例如渲染页的四个参数：`[输入类型 ▼] [场景 ▼] [Integrator ▼] [Sample Count 256]`，原本占用 4 行空间，现在只需 1-2 行。

### 3.2 控件体积压缩
- 输入框和按钮的上下 padding 从 `0.8rem` 缩减为 **`4px` 或 `6px`**。
- 采用更扁平的下拉框和单选框设计。

---

## 4. 任务状态与日志：全局浮窗化 (Global Drawer / Float Panel)

**这是释放核心屏幕空间的最重要改动。**
目前右侧固定的“任务状态/日志”面板（或在当前版本中通过按钮切换显示的列），占据了画廊极大的横向空间。日志和状态在“未执行任务”时是完全无用的信息。

### 4.1 底部可抽拉终端 (Bottom Terminal Drawer)
- 借鉴 IDE（如 VSCode）的设计，将日志和任务状态做成一个**底部抽屉（Drawer）**或**悬浮终端**。
- 默认高度仅为 `32px`（显示一行最新状态：`[Status: Running] Rendering alum-bronze (45%)...`）。
- 点击或上拉可展开至 `300px` 高度查看完整日志历史。
- **收益**：主界面的宽度 100% 释放给文件列表和画廊，无论任务是否在运行，画廊都可以保持最大宽度。

### 4.2 迷你进度与状态悬浮窗 (Floating Status Pill)
- 如果不使用底部抽屉，也可以在屏幕右下角做一个极简的悬浮面板。
- 包含进度条和关键状态指示灯，点击可展开详细日志流。

---

## 5. 画廊与数据列表的专业化

### 5.1 列表（Table/List）的表格化倾向
- 材质列表和模型运行记录的间距进一步收紧。
- 在 Models 页的 Runs 列表中，将大块的卡片（Card）转变为**高密度数据行（Data Row / Table）**。去掉卡片的底色和边框，只用浅色的 `border-bottom` 区分。
- 增加快速悬停反馈，不需要厚重的选中背景。

### 5.2 画廊（Gallery）的自适应缩放
- 增加一个全局的“缩放滑块”或“网格大小切换”，允许用户一键在“大图详细模式（每行 3-4 图）”和“超密集缩略图模式（每行 10 图以上）”之间切换。
- 去除图片下方不必要的冗余背景，让图片与文件名靠得更紧。

---

## 6. 额外体验优化建议 (Bonus Enhancements)

1. **骨架屏/占位符 (Skeleton Loading)**：在后端加载数据时，不要使用生硬的文字“正在读取...”，而是用微弱闪烁的低对比度块（Skeleton）保持界面布局稳定，防止加载完成后界面突然跳动。
2. **快捷键提示 (Hotkeys hints)**：在“启动渲染”、“全选”、“清空”等核心按钮旁，用浅色小字标注快捷键（如 `Ctrl+Enter`, `Ctrl+A`），鼓励键盘流操作。
3. **滚动条隐形 (Custom Scrollbars)**：Windows 默认的粗大滚动条在深色高密度界面中非常刺眼。需要自定义全局滚动条样式（`::-webkit-scrollbar`），使其变为细长、半透明的现代样式。

## 7. 实施路线图 (Implementation Plan)

- [ ] **Phase 1 (CSS 重构)**：调整 `styles.css`，修改所有的全局变量（`font-size`, `line-height`, `gap`, `padding`），快速实现空间压缩。
- [ ] **Phase 2 (表单重组)**：将 `RenderWorkbench` 和 `ModelsWorkbench` 中的表单改为网格或 Inline 布局，压缩高度。
- [ ] **Phase 3 (日志浮窗化)**：彻底移除右侧栏，封装一个新的 `<TerminalDrawer />` 组件，固定在屏幕底部或右下角，接入 WebSocket 日志。
- [ ] **Phase 4 (列表/表格优化)**：精简所有文件列表和运行记录项的 HTML 结构。
