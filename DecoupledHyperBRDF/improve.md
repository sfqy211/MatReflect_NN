# DecoupledHyperBRDF 项目改进方案

## 一、问题背景

1. **拟合目标冲突** — 单一网络统一拟合漫反射 + 镜面分量，二者动态范围、频率特性差异显著
2. **高频细节丢失** — 大面积漫反射主导损失优化，高频高光细节易被淹没、重建模糊
3. **稀疏采样稳定性不足** — 重建稳定性与精度不足

## 二、三分支架构设计

### 分支分工

| 分支 | 职责 |
|------|------|
| 漫反射分支 | 稳定拟合低频颜色基底 |
| 镜面分支 | 专项学习高频高光残差 |
| 门控分支 | 逐方向学习融合权重，控制镜面分量贡献 |

> 仅新增 **3 组独立超网络**，专门用于生成三分支架构所需的关键参数。

## 三、训练策略

### 阶段 A：稳定初始化

- 复用原有 Baseline checkpoint 做训练参考
- 门控分支 bias 初始化为负值（如 -2），初始门控输出极小
- 模型优先学习漫反射基底，锁定稳定基础

### 阶段 B：联合微调

- 逐步放大高光损失权重，强化高光细节拟合
- 后期逐步减弱门控正则，释放镜面分支能力

## 四、文件改动清单

| 文件 | 改动内容 |
|------|----------|
| `models.py` | 新增 DecoupledHyperBRDF 类，实现三分支架构 |
| `main.py` | 替换模型实例、扩展损失函数、新增分支损失日志 |
| `test.py` | 调整参数保存逻辑，输出三分支参数字典 |
| `pt_to_fullmerl.py` | 适配三分支推理，实现融合后 BRDF 导出 |

## 五、适配链路

```
.pt权重 → .fullbin格式 → Mitsuba渲染评估
```

### 最小改动适配

- **推理导出（test.py）**：单组 Hyponet 参数 → 三分支参数字典保存
- **格式转换（pt_to_fullmerl.py）**：单网络推理 → 三分支联合推理 + 融合输出
